# 豆豆 - AI 陪聊小助手 设计文档

## 概述

基于 LLM 的个人 AI 陪伴聊天 App。语音输入为主，流式文字回复 + TTS 语音播报，界面简单可爱。

目标平台：Android（Flutter 3.44.2）

## 角色形象

- 名字：豆豆
- 形象：一只紫色小胖鸟（🐤），固定头像
- 性格：温暖、活泼、用"啾~""咕咕~"等拟声词

## 视觉风格

自然萌趣 + 紫色点缀（方案 1：紫绿均衡）。

| 元素 | 样式 |
|------|------|
| 主题色 | 紫色系 `#ab47bc` / `#ce93d8` / `#b39ddb` |
| 辅助色 | 浅绿点缀 `#e8f5e9` |
| 背景 | 极浅紫 `#f5f0fa` |
| AI 气泡 | 白底 + 浅紫边框 `#e1bee7`，圆角 `20,20,20,6`，左边 + 小鸟头像 |
| 用户气泡 | 紫色底 `#ce93d8` 白字，圆角 `20,20,6,20`，右边 |
| 顶部栏 | 紫色渐变 `#b39ddb → #ce93d8`，白色文字 |
| AI 消息末尾 | 🔊 播放按钮（小圆角药丸） |
| 流式输出中 | 气泡内文字逐字出现 + 底部三个跳动 `●` |
| 语音按钮 | 80px 紫色圆形，居中，"按住说话"大字，录音时变红脉冲 |
| 输入框 | 白底 + 紫色描边圆角 |

## 交互流程

### 语音输入（主流程）
1. 按住 🎤 → 录音中（红圈脉冲动画）
2. 松开 → 显示"识别中..."
3. ASR 识别完成 → 用户文字气泡出现 → 自动发送给 LLM
4. AI 流式回复（文字逐 token 出现 + 跳动点动画）
5. 流结束 → 自动 TTS 播放整段回复 → 回到待命
6. 用户随时可点 🔊 手动重播

### 文字输入（辅助）
1. 点右下角 ⌨ → 切换到文字输入栏
2. 输入文字 + 点发送
3. 流程同语音模式步骤 4-6

### TTS 触发规则
- 默认：AI 回复完成后自动播
- 手动：点任意 AI 消息旁的 🔊 按钮
- 按标点分句合成（`.。!！?？\n`），避免全量 Token 都触发

## 架构

```
lib/
├── main.dart                   # MaterialApp + 主题 + 入口
├── theme.dart                  # ThemeData（紫白色系、圆角风格）
├── models/
│   └── message.dart            # Message(role, content, isStreaming)
├── services/
│   ├── llm_service.dart        # HTTP SSE 流式请求 → Stream<String>
│   ├── asr_service.dart        # 音频字节上传 → Future<String>
│   └── tts_service.dart        # WebSocket 逐句合成 → 播放
├── widgets/
│   ├── chat_bubble.dart        # 单条消息气泡（AI/用户双样式 + TTS 按钮）
│   ├── avatar_widget.dart      # 豆豆小鸟头像
│   ├── typing_dots.dart        # 流式输出跳动点动画
│   └── voice_input_button.dart # 语音按钮 + 按住录音逻辑
└── screens/
    └── chat_screen.dart        # 聊天主页面，持有所有状态
```

**外部依赖**：`dart:io`（HTTP/WebSocket）、`record`（录音）、`audioplayers`（播放 TTS 音频）、`permission_handler`（麦克风权限）。

## 状态管理

使用 `ChatScreen` 自身的 `StatefulWidget` + `setState`，零额外状态管理依赖。

```
ChatScreen._messages: List<Message>     # 消息列表
ChatScreen._isAiReplying: bool          # AI 回复中（控制动画）
ChatScreen._isRecording: bool           # 录音中（控制按钮样式）
ChatScreen._currentPlayingId: String?   # 当前播放 TTS 的消息 ID
```

数据流：`ASR识别 → 用户消息进列表 → LLM流式更新AI消息 → TTS播放`

## 错误处理

| 场景 | 表现 |
|------|------|
| 网络断开 | AI 气泡显示"网络断了，请重试" |
| ASR 识别失败 | SnackBar："没听清，再试一次？" |
| TTS 播放失败 | 静默跳过，聊天不受影响 |
| LLM 流中断 | 显示已收到的内容 + "..." |
| 麦克风权限拒绝 | 提示去设置开启权限 |
